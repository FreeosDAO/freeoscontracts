◆ claim()

void freeos::claim ( const name &  user)

claim action.

This action is run by the user to claim this week's allocation of freeos tokens.

Parameters
owner- the user account to execute the claim action for.
Precondition
Requires authorisation of the user account
The user must pass claim eligibility requirements:
  • must be registered as a freeos user (has a record in the users table)
  • must not have already claimed for the current week
  • must have staked the required amount of EOS tokens
  • must hold the required amount of freeos tokens.

const asset &  maximum_supply 
)

Create action.

Allows issuer account to create a token in supply of maximum_supply.

Parameters
issuer- the account that creates the token,
maximum_supply- the maximum supply set for the token created.
Precondition
Token symbol has to be valid,
Token symbol must not be already created,
maximum_supply has to be smaller than the maximum supply allowed by the system: 1^62 - 1.
Maximum supply must be positive;

If validation is successful a new entry in statstable for token symbol scope gets created.

void freeos::dereg ( const name &  user)

dereg action.

Creates a record for the user in the 'stakereqs' table.

Parameters
user- the account to be registered,
account_type- the type of account: "e" is EOS wallet user, "d" is Dapp Account user, "v" is Voice verified user.
Precondition
Requires permission of the contract account
The user account must be previously registered
The account must not have any staked EOS recorded

If validation is successful the record for the user account is deleted from the 'stakereqs' table.

void freeos::stake ( name  user,
name  to,
asset  quantity,
std::string  memo 
)

stake action.

Adds a record of staked EOS tokens in the user account record in the 'stakereqs' table.
This action is automatically invoked on the freeos account receiving EOS from the user.

Parameters
user- the account sending the EOS tokens to freeos,
to- the 'freeos' account,
quantity- the asset, i.e. an amount of EOS which is transferred to the freeos account,
memo- a memo describing the transaction
Precondition
The user account must be previously registered
The asset must be equal to the stake requirement for the user

If validation is successful the record for the user account is updated to record the amount staked and the date/time of the receipt.

◆ unstake()

void freeos::unstake ( const name &  user)

unstake action.

Transfer staked EOS back to the user if the waiting time of 1 week has elapsed since staking. Updates the record of staked EOS tokens in the user account record in the 'stakereqs' table.

Parameters
user- the account to transfer the previously EOS tokens to,
Precondition
The user account must be previously registered and has a non-zero balance of staked EOS

If transfer is successful the record for the user account is updated to set the amount staked and stake time/date to zero.
