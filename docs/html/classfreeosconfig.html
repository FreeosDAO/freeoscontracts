getthreshold action.

This action is intended for testing the 'stakereqs' table. It displays the stake requirement for the number of users and the account type.

Parameters
numusers- the lower bound of the range of user population (e.g. if the band is for 5,000-10,000 users, then enter 5000)
account_type- "e" for EOS wallet user, "d" for Dapp Account user, "v" for Voice verified user.

stakeupsert action.

This action creates a new entry, or modifies an existing entry, in the 'stakereqs' table.

Parameters
threshold- the lower bound of the range of user population (e.g. if 5,000-10,000 users require a stake of 10 EOS, then enter 5000). This is the promary key of the table,
value_e- the amount of EOS stake required for an EOS wallet user,
value_d- the amount of EOS stake required for a Dapp Account user,
value_e- the amount of EOS stake required for a Voice verified user,
value_e- (reserved for future use, enter 0)
Precondition
requires permission of the contract account
